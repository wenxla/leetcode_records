# 146. LRU Cache

> [LeetCode Link](https://leetcode.com/problems/lru-cache/) | Difficulty: **Medium** | Tags: Hash Table, Linked List, Design, Doubly-Linked List

## Problem

LRU: Least recently used cache
in cache, if this value has used recently, update its used. 
when cache full, remove the least recently used .

## Thinking Process
think as books stack. when the books is too much, move the bottom one; if you want to watch a book in here, move it to the top.
so, we use doubleLinkedList + hashmap to save.

==node: key, value, prev, tail==
==doubleLinkedList: node head, node tail==
==hashmap: key, node==
